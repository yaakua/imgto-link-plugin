# FaFaFa-全部发 商店上架文案（中文）

## 插件名称
FaFaFa-全部发

## 商店摘要（短描述）
从 fafafa.ai mdeditor 一键发布到多个内容平台，自动检测登录状态并尽可能保留排版。

## 副标题 / 宣传短句（可选）
- 一次编辑，多平台自动发布
- 在 fafafa.ai 里直接调用浏览器插件发布
- 自动检测登录状态，降低重复复制成本

## 商店关键词（可选）
Markdown、多平台发布、自动发布、内容分发、公众号排版、CSDN、掘金、小红书、浏览器扩展、fafafa.ai

## 详细描述
FaFaFa-全部发 是 fafafa.ai 官方浏览器扩展，面向需要把一篇 Markdown 内容同步到多个平台的创作者、运营和技术作者。

安装后，你可以直接在 fafafa.ai 的 mdeditor 中点击“发布”，检测已登录的平台账号，选择目标平台，然后让插件自动打开对应编辑器并填充内容。

核心能力：
- 一次编辑，多平台分发
- 自动检测平台登录状态
- 支持富文本 / Markdown / 微信样式内容桥接
- 尽可能保留排版效果，降低二次整理成本
- 本地浏览器内执行，不依赖插件开发者服务器转发正文内容

适用场景：
- 技术博客多平台同步
- 内容运营批量分发
- 微信公众号 / 小红书 / 掘金 / CSDN 等平台草稿同步

## 面向审核的说明
- 插件仅在用户主动点击“发布”后运行，不会后台自动代发
- 正文内容默认在本地浏览器中处理，不通过插件开发者服务器中转
- 站点权限仅覆盖 fafafa.ai 与目标发布平台，用于登录检测、打开编辑器、自动填充内容
- 如商店审核要求，可在备注中补充 `debugger` 与 `clipboardRead` 权限用于富文本粘贴与兼容复杂编辑器

## 权限说明
- `scripting`：向目标平台编辑器注入内容
- `cookies`：检查目标平台登录状态
- `tabGroups` / `activeTab`：组织和管理自动打开的发布标签页
- `debugger`：部分平台需要模拟粘贴行为以保留排版
- `clipboardRead`：读取富文本 HTML 内容用于同步
- `storage`：缓存必要的本地状态
- `offscreen` / `declarativeNetRequest`：支持某些平台的登录检测与兼容流程

## 隐私摘要
- 不收集个人信息
- 不将正文内容上传到插件开发者服务器
- 仅在你主动触发发布时访问目标平台
