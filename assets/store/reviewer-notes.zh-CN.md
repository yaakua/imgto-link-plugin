# FaFaFa-全部发 审核备注模板（中文）

> 用于 Chrome Web Store / Edge Add-ons 提交时的“给审核团队的备注”。

## 标准版

FaFaFa-全部发 是 `fafafa.ai` 的官方浏览器扩展，用于在用户主动点击“发布”后，把 mdeditor 中的内容同步到多个内容平台。

插件运行方式说明：

1. 仅当用户访问 `https://fafafa.ai/*` 或受支持的目标发布平台时，扩展脚本才会运行。
2. 仅当用户在编辑器中主动点击“发布”时，插件才会开始检测登录状态、打开目标编辑器并填充内容。
3. 正文内容默认在本地浏览器中处理，不会上传到插件开发者服务器。
4. 扩展不会在后台自动发布内容，也不会自动代表用户提交文章。

权限用途说明：

- `scripting`：向目标平台编辑器填充文章内容
- `cookies`：检查目标平台登录状态
- `tabGroups` / `activeTab`：管理发布过程中打开的标签页
- `debugger`：个别富文本编辑器需要模拟粘贴操作以保留排版
- `clipboardRead`：读取 HTML 富文本以支持排版保留
- `storage`：保存必要本地状态
- `offscreen` / `declarativeNetRequest`：支持兼容性处理和部分登录检测流程

官方说明页面：

- 安装说明：https://fafafa.ai/publish-extension
- 编辑器入口：https://fafafa.ai/mdeditor

## 如果被问“为什么需要 debugger 权限？”

我们在部分富文本编辑器中需要模拟更接近真实用户粘贴的行为，才能尽可能保留 HTML 排版结构。该权限仅在用户主动发起发布流程时使用，不会用于浏览器调试、页面抓取或后台监控。

## 如果被问“为什么需要 cookies 权限？”

该权限仅用于检测用户是否已登录受支持的平台，从而避免打开目标页面后出现未登录状态导致的失败。我们不会读取与发布无关的敏感账户信息，也不会把 cookie 上传到外部服务器。
